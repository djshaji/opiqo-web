Privacy Policy

Effective date: April 7, 2026  ·  Opiqo for Android, Windows & Linux

TL;DR — The Short Version

Opiqo processes all audio locally on your device. It does not send your recordings, presets, or usage data to any remote server. There is absolutely no data collection, no advertising, no analytics, and no crash-reporting SDK in this app.

I am one of the old-school developers from the era before everything was a subscription. I built Opiqo to be a simple way for guitarists like me to use these plugins and get amazing tones. This app is for the guitarist struggling to find that first gig — I feel you. Keep on rocking. All audio processing and file handling happens locally on your device. You can inspect the open-source code on GitHub to see exactly how the app works.

1. Overview

Opiqo is an LV2 plugin host and guitar effects processor. It requests microphone access through the RECORD_AUDIO permission so it can capture input audio for live processing and recording. The app does not include advertising SDKs, analytics SDKs, or crash-reporting SDKs.

2. Information the App Can Access

  • Microphone audio — when you grant microphone permission and start audio processing or recording.
  • Files you explicitly choose — through Android's document picker for preset import or plugin-related file selection.
  • Files created by the app — such as recordings and saved presets stored on your device.

3. Why Microphone Access Is Requested

Opiqo is a guitar and audio effects processor. Microphone access is required so the app can capture input audio, pass it through the LV2 effects chain, and optionally record the processed output. Without this permission, the core audio-processing features will not work.

4. How Audio and Files Are Handled

  • Audio is processed locally on the device through the app's native C++ engine.
  • Recordings are stored on-device in the format you choose (WAV, FLAC, MP3, Opus, OGG Vorbis).
  • Presets are stored locally and can be exported to a JSON file when you initiate an export.
  • Imported presets are read from a file you select and copied into the app's presets directory for immediate use.
  • Plugin-related files you select may be copied into the app's internal storage so the associated LV2 plugin can reference them.

5. Data Sharing

Opiqo does not send your audio recordings, presets, or usage data to any remote service as part of normal operation. If you export files, those files go to the location you choose using your device's system file picker — no intermediary server is involved.

6. Third-Party Components

Opiqo uses the following third-party open-source libraries to support local audio processing and LV2 hosting. None of these components transmit data externally on behalf of this app:

  • Google Oboe — low-latency audio API for Android
  • Lilv, Serd, Sord, Sratom, Zix — LV2 plugin discovery and hosting
  • JACK components — audio connectivity (Linux)

This policy does not replace their own licenses or documentation.

7. Open-Source Transparency

Opiqo is open source and released under the MIT License. You can inspect the full implementation at github.com/djshaji/opiqo-multi. If the app's behavior changes in future releases, this policy will be updated to reflect those changes.

8. Your Choices

  • You can deny microphone permission, although core audio-processing features will not work without it.
  • You choose whether to record audio, export presets, import presets, or select plugin-related files.
  • You can remove recordings or exported files from your device using your normal file-management tools.

9. Changes to This Policy

This page will be updated if the app's permissions, data flows, or third-party services change. For the latest project status, review the repository and release notes on GitHub.

10. Contact

For questions, bug reports, or privacy-related concerns about Opiqo, please use the issue tracker on GitHub.
