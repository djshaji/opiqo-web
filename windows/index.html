Windows-specific Capabilities

Built for the Windows Desktop

Every component designed around Windows audio conventions and Win32 best practices. The shared DSP core (LiveEffectEngine) runs unchanged from the Linux build — only the audio and UI platform layers are Windows-specific.

WASAPI Shared & Exclusive

Full-duplex WASAPI streams in both shared and exclusive mode. Exclusive mode gives you the lowest possible latency by bypassing the Windows audio engine mix stage entirely.

Win32 Native UI

Full Win32 desktop interface with a 2×2 plugin slot grid. Menu bar, status bar, resizable layout, and all controls built with the native Windows API — no third-party UI frameworks required.

MMCSS Real-time Priority

The WASAPI audio thread runs under MMCSS (Multimedia Class Scheduler Service) at real-time priority. The processing loop is fully allocation-free for glitch-free performance under load.

LV2 Plugin Host

Full LV2 host using Lilv. 54 bundled guitar effects included out of the box. Browse and load plugins per slot with dynamic parameter controls generated at runtime from plugin metadata.

Hot-plug Device Detection

IMMNotificationClient detects USB audio device arrivals and removals in real time, refreshing the device dropdown and recovering the audio stream automatically without requiring an app restart.

AppData Settings Storage

Settings stored at %APPDATA%\Opiqo\settings.json. Persists selected input/output device IDs, sample rate, block size, share mode, recording format, and gain across sessions.

Multi-format Recording

Record the post-FX signal to WAV, MP3, or OGG. Format and quality selection are surfaced directly in the ControlBar. Recording runs concurrently while all 4 effect slots are active.

Preset Import / Export

Export your full pedalboard configuration to a JSON file. Import on any device running Opiqo — presets are compatible across Android, Windows, and Linux builds.

Settings Dialog

Configure sample rate, block size, and WASAPI share mode from a dedicated settings dialog. Changes are applied immediately — the engine stops, reconfigures, and restarts seamlessly with the new settings.

Architecture

Three-Layer Design

The Windows build is cleanly separated into Win32 UI, WASAPI platform audio, and shared DSP layers. Only the top two layers are Windows-specific; the DSP core is shared verbatim with the Linux and Android builds.

Win32 UI Layer

MainWindow, PluginSlot, ControlBar, PluginDialog, SettingsDialog. All window messages and user actions handled on the UI thread.

Audio Platform Layer

AudioEngine owns COM/WASAPI resources and the dedicated real-time audio thread. WasapiDeviceEnum handles endpoint discovery and change notifications via IMMNotificationClient.

DSP Host Layer (Shared)

LiveEffectEngine + LV2Plugin are shared verbatim with the Linux and Android builds. No platform-specific code in the DSP or file-writing layer.

Signal Flow

  1. 01

    WASAPI Captures Audio

    WASAPI captures audio from the selected input endpoint (USB interface, microphone, etc.) and delivers it as float32 or PCM frames to the dedicated real-time audio thread.

  2. 02

    LiveEffectEngine Processes

    LiveEffectEngine::process(input, output, frames) runs each active LV2 plugin in slot order. Buffer conversion handles the WASAPI format ↔ float32 path as needed — no allocations in the hot path.

  3. 03

    WASAPI Renders Processed Signal

    The processed buffer is written to the WASAPI render endpoint. Simultaneous recording writes the same buffer to the output file via FileWriter in WAV, MP3, or OGG format.

  4. 04

    Win32 UI Reflects State

    Engine state is surfaced to the UI thread via posted window messages. Device changes from IMMNotificationClient are marshalled back to the UI thread and reflected in the status bar and device dropdowns.

Plugin Interface

2×2 Plugin Slot Grid

Four independent plugin slots in a resizable 2×2 Win32 grid. Each slot independently manages its own plugin instance, bypass state, and parameter panel.

Plugin Slot Controls

Each plugin slot provides a consistent set of per-slot controls:

  • Add plugin — Opens the plugin browser dialog backed by getAvailablePlugins()
  • Bypass / Enable — Toggles the loaded plugin on or off in real-time
  • Delete plugin — Removes the current plugin and clears the parameter panel
  • Parameter panel — Dynamic Win32 controls generated from LV2Plugin port metadata

Dynamic Parameter Controls

  • Float (control range) → Slider / trackbar
  • Float (toggle) → Checkbox
  • Float (enumeration) → ComboBox with scale-point labels
  • Atom file path → Browse button with file open dialog

ControlBar

  • Power toggle — Starts / stops the WASAPI stream; the single source of truth for engine transport
  • Gain slider — Master input gain
  • Record button — Starts / stops post-FX recording
  • Format dropdown — WAV / MP3 / OGG; quality selector shown only for lossy formats
  • Device selectors — Input and output endpoint dropdowns in the status area
System Requirements

What You Need to Run Opiqo

Opiqo Windows runs on any 64-bit Windows 10 or later installation. No drivers or additional runtimes required for the basic build.

Runtime Requirements

  • Windows 10 or later (64-bit)
  • WASAPI (built into Windows Vista+)
  • Any WDM-compatible audio interface
  • Visual C++ Redistributable (MSVC build)
  • No additional drivers needed for shared mode

Build Requirements

  • CMake ≥ 3.22
  • MinGW-w64 or MSVC with C++17
  • Lilv (LV2 plugin loader)
  • libsndfile ≥ 1.0
  • libmp3lame (for MP3 recording)
  • libopus / libopusenc (for Opus recording)

Recommended Audio Setup

  • USB audio interface (vs. built-in audio)
  • WASAPI exclusive mode for lowest latency
  • Buffer size: 256 samples at 48 kHz
  • Disable Windows audio enhancements for the device
  • Set audio device to 48 kHz in Sound Control Panel
