§1

Objective

Build a native Linux port of the Opiqo LV2 plugin host that runs low-latency duplex audio via the JACK Audio Connection Kit, processes audio through the existing LiveEffectEngine in real time, provides a GTK4 desktop UI with 4 plugin slots in a 2×2 layout, and supports JACK port routing, transport control, and recording.

Primary Runtime Path

JACK capture ports
  → LiveEffectEngine::process(input, output, frames)
    → JACK playback ports

The DSP and file-writing layer (LiveEffectEngine, LV2Plugin, FileWriter, LockFreeQueue) is shared verbatim with the Windows build. Only the audio platform and UI layers need a new Linux-specific implementation.

Why JACK instead of PipeWire-native or ALSA?

  • JACK provides a guaranteed real-time callback (jack_process_callback) with float32 buffers matching LiveEffectEngine's expected format — zero conversion required.
  • PipeWire ships a full JACK compatibility layer (pipewire-jack); the same binary works on both classic JACK2 and modern PipeWire systems.
  • ALSA would require manual latency management and ring-buffer design already solved by JACK.

Build System

A new linux-default CMake preset and target block alongside the existing Windows preset.

CMakePresets.json — new preset

{
  "name": "linux-default",
  "displayName": "Linux (GTK4 + JACK)",
  "binaryDir": "${sourceDir}/build-linux",
  "cacheVariables": {
    "CMAKE_BUILD_TYPE": "Release",
    "OPIQO_TARGET_PLATFORM": "linux"
  }
}

CMakeLists.txt additions

if(OPIQO_TARGET_PLATFORM STREQUAL "linux")
  find_package(PkgConfig REQUIRED)
  pkg_check_modules(GTK4   REQUIRED gtk4)
  pkg_check_modules(JACK   REQUIRED jack)
  pkg_check_modules(LILV   REQUIRED lilv-0)
  pkg_check_modules(SNDFILE REQUIRED sndfile)

  add_executable(opiqo
    src/main_linux.cpp
    src/linux/AppSettings.cpp
    src/linux/AudioEngine.cpp
    src/linux/JackPortEnum.cpp
    src/linux/MainWindow.cpp
    src/linux/ControlBar.cpp
    src/linux/PluginSlot.cpp
    src/linux/PluginDialog.cpp
    src/linux/ParameterPanel.cpp
    src/linux/SettingsDialog.cpp
    src/LiveEffectEngine.cpp
    src/FileWriter.cpp
    src/LockFreeQueue.cpp
  )

  target_link_libraries(opiqo PRIVATE
    ${GTK4_LIBRARIES} ${JACK_LIBRARIES}
    ${LILV_LIBRARIES} ${SNDFILE_LIBRARIES}
    pthread dl
  )
endif()

Build Commands

git clone https://github.com/djshaji/opiqo-linux.git
cd opiqo-linux
cmake --preset linux-default
cmake --build build-linux -j$(nproc)
./build-linux/opiqo
§5

Architecture

Three clearly separated layers with explicit thread ownership and lock-free cross-thread communication.

┌─────────────────────────────────┐ │ GTK4 UI Layer │ │ MainWindow, PluginSlot, │ │ ControlBar, PluginDialog, │ │ ParameterPanel, SettingsDialog │ │ Thread: GTK main thread │ └────────────────┬────────────────┘ │ calls (g_idle_add for replies) ┌────────────────▼────────────────┐ │ Audio Platform Layer │ │ AudioEngine (JACK client), │ │ JackPortEnum │ │ Thread: JACK real-time thread │ └────────────────┬────────────────┘ │ calls (LockFreeQueue for params) ┌────────────────▼────────────────┐ │ DSP Host Layer │ │ LiveEffectEngine + LV2Plugin │ │ (shared with Windows build) │ └─────────────────────────────────┘

Thread Ownership

  • GTK main thread — all widget creation, signal handlers, and UI state changes
  • JACK real-time thread — jack_process_callback only; runs LiveEffectEngine::process()
  • Cross-thread — parameter updates via LockFreeQueue; JACK events to UI via g_idle_add()

State Machine

  • Off → Starting → Running (Power toggle on)
  • Running → Stopping → Off (Power toggle off)
  • Any → Error on JACK failure or server loss
  • State polled every 200 ms via g_timeout_add
  • JACK server loss via jack_set_server_lost_callback

AudioEngine Public API

enum class State {
  Off, Starting, Running,
  Stopping, Error
};

bool start(const std::string& capturePort,
           const std::string& playbackPort);
void stop();
State       state()        const;
int32_t     sampleRate()   const;
int32_t     blockSize()    const;
std::string errorMessage() const;
uint64_t    xrunCount()    const;
§6

Milestone Plan

11 milestones ordered to prove audio first, then build UI on top. Each milestone has concrete deliverables and acceptance criteria.

M0

Project Skeleton and Build Wiring

Compile a minimal GTK4 window under the new linux-default preset.

  • Add the linux-default preset to CMakePresets.json
  • Add the if(OPIQO_TARGET_PLATFORM STREQUAL "linux") block to CMakeLists.txt
  • Create src/main_linux.cpp with a GtkApplication that opens a blank GtkApplicationWindow
  • Add stub header/source files for all src/linux/ modules

Acceptance Criteria

  • cmake --preset linux-default && cmake --build build-linux succeeds without warnings on GTK4 ≥ 4.6 and JACK ≥ 1.9
  • Binary runs and shows a window without crashes
M1

JACK Port Enumeration and Settings Model

Discover available JACK client ports and persist user preferences.

  • JackPortEnum: open a temporary JACK client for probing; implement enumerateCapturePorts() and enumeratePlaybackPorts() using jack_get_ports() with JackPortIsPhysical
  • Return PortInfo { id, friendlyName, isDefault } list matching shared DeviceInfo interface
  • Register jack_set_port_registration_callback(); marshal change events to GTK thread via g_idle_add()
  • AppSettings: fields capturePort, playbackPort, sampleRate, recordFormat, recordQuality, gain
  • Load from $XDG_CONFIG_HOME/opiqo/settings.json (fallback to ~/.config/opiqo/settings.json); save on shutdown

Acceptance Criteria

  • Port list refreshes when a device is plugged/unplugged
  • Missing saved port falls back to first physical capture/playback port
M2

AudioEngine Core (JACK, Pass-Through)

Prove full-duplex JACK pipeline with a clean pass-through path.

  • jack_client_open("opiqo", JackNoStartServer, nullptr)
  • Register one capture port and one playback port via jack_port_register
  • Implement jack_process_callback: copy input → output (pass-through, no DSP yet)
  • Register jack_set_xrun_callback to increment xrunCount_
  • start() calls jack_activate() then jack_connect() to selected physical ports
  • stop() calls jack_deactivate() then jack_client_close()

Acceptance Criteria

  • 15-minute pass-through run without xruns on a quiet system
  • 50 repeated start()/stop() cycles without leaked JACK clients
M3

LiveEffectEngine Integration in JACK Callback

Replace pass-through with the real DSP path.

  • In jack_process_callback, call engine_->process(capture_buf, playback_buf, nframes)
  • JACK native format is float (jack_default_audio_sample_t) — no format conversion needed
  • Register a second capture and playback port pair for stereo, or remix mono to both channels
  • Validate bypass: with no plugins loaded, signal passes through cleanly

Acceptance Criteria

  • With no plugins loaded: clean pass-through audible
  • With a plugin loaded in slot 1: effect audible on output
  • No allocations in the JACK callback (verified with MALLOC_CHECK_=3)
M4

Power Toggle State Machine

Make the Power button the single source of truth for engine transport.

  • Implement atomic State variable with transitions: Off → Starting → Running, Running → Stopping → Off, any → Error
  • Wire ControlBar power toggle: On → AudioEngine::start(), Off → AudioEngine::stop()
  • g_timeout_add(200, pollEngineState, this) polls state on the GTK main thread
  • On State::Error: revert toggle to Off, show error in status bar
  • jack_set_server_lost_callback → move to Error via g_idle_add

Acceptance Criteria

  • Rapid start/stop spam (50×) leaves engine in a consistent state
  • JACK server crash moves UI to Error state without hang
M5

MainWindow Layout and 2×2 Plugin Slot Grid

Build the full GTK4 shell layout.

  • Widget hierarchy: GtkApplicationWindow → GtkBox → GtkHeaderBar + GtkGrid (2×2) + ControlBar + statusBar_
  • Define static skeleton in main_window.ui GtkBuilder file; load via gtk_builder_new_from_resource
  • Minimum window size: 900 × 650 via gtk_window_set_default_size
  • Load opiqo.css via GtkCssProvider for slot framing

Acceptance Criteria

  • Window resizes gracefully; slots scale proportionally
  • Layout renders correctly on both X11 and Wayland
M6

Plugin Management UI

Connect plugin lifecycle controls to existing LiveEffectEngine APIs.

  • PluginSlot: one GtkFrame per slot with header row (name label + Add + Bypass + Delete) and parameter panel area
  • PluginDialog: GtkListView + GtkSingleSelection backed by getAvailablePlugins(); GtkSearchEntry + GtkFilterListModel for filtering
  • Add (+): calls addPlugin(slot, uri) and rebuilds ParameterPanel
  • Bypass: calls setPluginEnabled(slot, !bypassed)
  • Delete (×): calls deletePlugin(slot) and clearParameterPanel()

Acceptance Criteria

  • Each of the 4 slots independently manages its plugin instance
  • Bypass toggles effect in real time with no audible glitch
M7

Dynamic Parameter Panels

Generate GTK4 controls at runtime from LV2Plugin::PortInfo.

  • ParameterPanel: a GtkScrolledWindow containing a GtkBox (vertical) with one row per port
  • Float (range) → GtkScale; Float (enum) → GtkDropDown; Toggle → GtkCheckButton; Trigger → GtkButton; AtomFilePath → GtkFileChooserNative
  • Value changes connect to engine_->setValue() via g_signal_connect
  • All callbacks fire on the GTK main thread — no marshalling needed

Acceptance Criteria

  • Slider movements update sound in real time without JACK xruns
  • Panels scroll cleanly when a plugin exposes many ports
  • Rebuilding the panel leaves no dangling widget references
M8

Recording UX and Format Handling

Wire recording controls to FileWriter via LiveEffectEngine.

  • Record toggle: on → engine_->startRecording(fd, format, quality); off → stopRecording()
  • Output path: g_get_user_special_dir(G_USER_DIRECTORY_MUSIC) + timestamp filename
  • Format dropdown bound to AppSettings::recordFormat (0=WAV, 1=MP3, 2=OGG)
  • Quality dropdown shown only for lossy formats; hidden for WAV

Acceptance Criteria

  • Recorded WAV/MP3/OGG files open in external players without corruption
  • Recording can start and stop while all 4 plugin slots are active
  • File is finalized cleanly even if the app exits during recording
M9

Settings Dialog and Runtime Reconfiguration

Configurable audio preferences and preset import/export.

  • SettingsDialog: GtkDialog with two tabs via GtkNotebook
  • Audio tab: JACK capture/playback port dropdowns (from JackPortEnum), read-only sample rate and buffer size labels, Apply button (stop → apply → restart)
  • Presets tab: Export → GtkFileChooserNative save writes getPresetList() JSON; Import → reads JSON and calls applyPreset() per slot
  • Settings saved via AppSettings::save() on dialog confirm

Acceptance Criteria

  • Port selection changes survive app restart
  • Import of a preset restores all four slot configurations
  • Changing the port while the engine is running restarts seamlessly
M10

Hardening, Performance, and Release Readiness

Production stability and diagnosability.

  • Set JACK thread to SCHED_FIFO real-time priority via jack_client_open flags or pthread_setschedparam
  • Add xrun counter in the status bar, updated via the 200 ms GTK poll
  • Log xruns and engine errors to ~/.local/share/opiqo/opiqo.log using logging_macros.h
  • Device hot-plug stress test: plug/unplug USB audio device 20× while running
  • Verify JACK callback is allocation-free with Valgrind --tool=massif and AddressSanitizer
  • PipeWire-JACK compatibility verification with pipewire-jack shim

Acceptance Criteria

  • 60-minute soak run without xruns on a quiet system
  • JACK server restart recovers engine without app restart
  • Zero memory leaks reported by Valgrind on clean startup/shutdown
§7

Cross-cutting Technical Requirements

Real-time Safety (JACK Callback)

  • No heap allocations inside jack_process_callback
  • No mutexes that can block; use LockFreeQueue for plugin parameter updates
  • No GTK calls; communicate back to UI via g_idle_add()
  • Preallocate all conversion and channel-split buffers in AudioEngine::start()

Thread Safety

  • GTK main thread: all widget creation, signal handlers, UI state
  • JACK real-time thread: jack_process_callback only
  • std::atomic<State> for engine state visible to both threads
  • LockFreeQueue for parameter values (main → RT direction)

Error Handling

  • Surface all JACK errors in the status bar via g_idle_add
  • jack_set_server_lost_callback: move to State::Error, show message, revert Power toggle
  • AppSettings::load() returns safe defaults on any file error
  • Xrun count displayed in status bar; logged to opiqo.log

Wayland / X11 Compatibility

  • Use gtk_file_chooser_native_new() for all file dialogs (required for Wayland portals)
  • Avoid GdkX11 and GdkWayland backend-specific APIs
  • Use gdk_display_get_default() only for font/DPI queries
  • Test UI renders correctly under both X11 and Wayland sessions

Test Plan

Functional Tests

  • App startup with valid and missing saved JACK ports
  • Power toggle on/off cycles (50 repetitions)
  • JACK port switch while running
  • Add/delete plugins in all 4 slots
  • Parameter changes under active audio
  • Recording for WAV, MP3, and OGG while all slots are active

Robustness Tests

  • USB audio device unplug/replug while JACK graph is active
  • Rapid Power toggle spam (100×)
  • JACK server kill and restart while engine is Running
  • Open Settings and Apply repeatedly during idle and Running states

Performance Tests

  • CPU usage: no plugins, 1 plugin, 4 plugins
  • Xrun count over 10 minutes at 48 kHz / 256-frame buffer
  • Latency: compare 256-frame vs 1024-frame buffer sizes
  • 60-minute soak run (release readiness)
  • Valgrind --tool=massif for JACK callback allocations

Prerequisites

Install these packages before building. Ensure JACK is running before launching Opiqo (or use PipeWire's JACK compatibility layer).

Debian / Ubuntu

sudo apt install \
  libgtk-4-dev \
  libjack-jackd2-dev \
  liblilv-dev \
  libsndfile1-dev \
  libmp3lame-dev \
  libopus-dev \
  libopusenc-dev \
  cmake make pkg-config

Fedora / RHEL

sudo dnf install \
  gtk4-devel \
  jack-audio-connection-kit-devel \
  lilv-devel \
  libsndfile-devel \
  lame-devel \
  opus-devel \
  opusenc-devel \
  cmake make pkg-config

PipeWire Setup (Modern Distros)

# Most modern distros ship PipeWire
# with the JACK compatibility layer.
# Verify it is active:
pw-jack --version

# Or install explicitly:
sudo apt install pipewire-jack    # Debian
sudo dnf install pipewire-jack-audio-connection-kit  # Fedora

# Run opiqo via pw-jack if needed:
pw-jack ./build-linux/opiqo

Sequencing Recommendation

  1. M0–M3 — Build wiring → JACK pass-through → DSP integration (audio first)
  2. M4–M6 — Transport state machine → shell layout → plugin management
  3. M7–M9 — Parameter UI → recording → settings
  4. M10 — Hardening and release readiness
