Linux-specific Capabilities

Built for the Linux Desktop

Every component designed with Linux audio conventions and GTK4 best practices. The shared DSP core (LiveEffectEngine) runs unchanged from the Windows build — only the audio and UI platform layers are Linux-specific.

JACK / PipeWire Audio

Full-duplex audio via JACK Audio Connection Kit. Works identically on classic JACK2 and modern PipeWire systems via the pipewire-jack compatibility shim — no configuration required.

GTK4 Native UI

Full GTK4 desktop interface with a 2×2 plugin slot grid built using GtkGrid. Layout defined in GtkBuilder XML (.ui files) and styled via GtkCssProvider.

SCHED_FIFO Real-time Thread

The JACK audio callback runs at SCHED_FIFO real-time priority, fully allocation-free and mutex-free. Parameter updates flow through a LockFreeQueue to avoid any priority inversion.

LV2 Plugin Host

Full LV2 host using Lilv. 54 bundled guitar effects included out of the box. Browse and filter plugins by name in the GtkListView-based plugin dialog.

X11 & Wayland Compatible

Uses only portable GTK4 APIs. No backend-specific GdkX11 or GdkWayland calls. File dialogs use GtkFileChooserNative for full Wayland / Flatpak portal compatibility.

XDG-Compliant Settings

Settings stored at ~/.config/opiqo/settings.json following the XDG Base Directory Specification. Includes capture/playback port, recording format, quality, and gain.

Multi-format Recording

Record the post-FX signal to WAV, MP3, or OGG via FileWriter and LiveEffectEngine. Output saved to ~/Music/ with a timestamp filename. Recording works while all 4 slots are active.

Hot-plug Device Detection

JACK port registration callbacks (jack_set_port_registration_callback) refresh available capture and playback ports automatically when a USB audio device is connected or disconnected.

Preset Import / Export

Export your full pedalboard configuration to a JSON file via GtkFileChooserNative. Import on any device running Opiqo — presets are shared across Android, Windows, and Linux.

Architecture

Three-Layer Design

The Linux build is cleanly separated into GTK4 UI, JACK platform audio, and shared DSP layers. Only the top two layers are Linux-specific; the DSP core is shared verbatim with the Windows build.

GTK4 UI Layer

MainWindow, PluginSlot, ControlBar, PluginDialog, ParameterPanel, SettingsDialog. All widgets run on the GTK main thread.

Audio Platform Layer

AudioEngine manages the JACK client lifecycle. JackPortEnum discovers and monitors physical audio ports. The jack_process_callback runs at SCHED_FIFO priority.

DSP Host Layer (Shared)

LiveEffectEngine + LV2Plugin are shared verbatim with the Windows build. The JACK float32 buffer format matches LiveEffectEngine::process() exactly — no conversion needed.

Signal Flow

  1. 01

    JACK Captures Audio

    JACK captures audio from the selected physical input port (e.g. system:capture_1) and delivers it as float32 frames to the registered process callback.

  2. 02

    LiveEffectEngine Processes

    LiveEffectEngine::process(input, output, frames) runs each active LV2 plugin in slot order. Parameter updates from the UI are dequeued from LockFreeQueue — allocation-free.

  3. 03

    JACK Plays Processed Signal

    The processed buffer is written to the JACK playback port and connected to system:playback_1/2. Simultaneous recording writes the same buffer to the output file via FileWriter.

  4. 04

    GTK4 UI Reflects State

    Engine state is polled every 200 ms on the GTK main thread via g_timeout_add. JACK events (xruns, server loss) are marshalled back to the UI thread with g_idle_add.

Plugin Interface

2×2 Plugin Slot Grid

Four independent plugin slots arranged in a 2×2 grid. Each slot manages its own plugin instance, bypass state, and parameter panel.

Plugin Slot Controls

Each of the four GtkFrame-based plugin slots provides a consistent set of controls:

  • Add (+) — Opens the plugin browser dialog to load an LV2 effect
  • Bypass toggle — Enables or disables the loaded plugin in real-time
  • Delete (×) — Removes the current plugin and clears the parameter panel
  • Parameter panel — Dynamic GTK4 controls auto-generated from LV2Plugin::PortInfo

Dynamic Parameter Controls

  • Float (range) → GtkScale horizontal slider
  • Float (enumeration) → GtkDropDown with scale-point labels
  • Toggle → GtkCheckButton
  • Trigger → GtkButton
  • File path → GtkButton "Browse…" using GtkFileChooserNative
System Requirements

What You Need to Run Opiqo

Opiqo Linux requires a reasonably modern distribution. It works on both classic JACK2 setups and modern PipeWire-based distros.

Runtime Dependencies

  • GTK4 ≥ 4.6
  • JACK ≥ 1.9 or PipeWire ≥ 0.3 with pipewire-jack
  • Lilv (LV2 plugin loader)
  • libsndfile ≥ 1.0
  • libmp3lame (for MP3 recording)
  • libopus + libopusenc (for Opus recording)
  • Linux kernel with SCHED_FIFO support
  • X11 or Wayland compositor

Build Dependencies

  • CMake ≥ 3.22
  • GCC ≥ 11 or Clang ≥ 14 with C++17
  • pkg-config
  • libgtk-4-dev
  • libjack-jackd2-dev
  • liblilv-dev
  • libsndfile1-dev
  • libmp3lame-dev, libopus-dev, libopusenc-dev

Recommended Audio Setup

  • USB audio interface (low-latency preferred)
  • JACK / PipeWire buffer size: 256 frames at 48 kHz
  • Real-time kernel or realtime-privileges group membership
  • PulseAudio / PipeWire bridge running (for other apps)
  • ALSA → JACK bridge not required; JACK connects directly
