Android Capabilities

Studio-Grade Effects on Android

Every component optimised for mobile audio performance. The same shared DSP core runs on Windows and Linux — only the audio and UI platform layers are Android-specific.

Google Oboe Low-Latency Audio

Full-duplex audio via Google's Oboe library (FullDuplexPass) built on top of AAudio or OpenSL ES depending on the device. Targets the lowest achievable round-trip latency on Android hardware.

Full LV2 Plugin Host

Complete LV2 host using Lilv, Sord, Sratom, and JACK libraries compiled for Android. 54 studio-quality guitar effects bundled out of the box. LV2 buf-size compliant — correctly advertises block length options sourced from the real Oboe burst size at runtime.

Swipeable 4-Pedal UI

Four independent pedal tabs in a swipeable ViewPager2 layout. Each slot hosts one LV2 plugin with its own bypass toggle, parameter sliders, and delete button. Plugin parameter controls are generated dynamically from LV2 port metadata.

Audio Device Selection

Separate input and output device spinners let you choose from every connected audio interface — USB audio interfaces, Bluetooth headsets, or the built-in microphone and speaker. Master volume slider scales the final output gain.

Multi-Format Recording

Record the post-FX signal in WAV (32-bit), FLAC (lossless), MP3, Opus, or OGG Vorbis. Recordings capture the fully processed signal — after all active pedal effects and master volume — and are saved to device storage.

Preset Backup & Restore

Export all saved presets to a single JSON file via the system document picker. Import from any readable JSON file and presets are written back to the app's presets directory and reloaded immediately — no restart required.

Real-time Parameter Control

Every LV2 plugin control port gets a labelled slider set to the plugin's default value. Range and label come directly from the plugin's LV2 metadata. Changes take effect in the next audio processing cycle — no dropouts.

Native C++ DSP Core

The LiveEffectEngine, LV2Plugin, and FileWriter are written entirely in C++ and compiled with the Android NDK for arm64-v8a, armeabi-v7a, x86, and x86_64 ABIs. A JNI bridge in jni_bridge.cpp connects the Java UI to the native engine.

Atom Messaging for File Paths

Plugins that accept external file paths (IR loaders, cab sims) receive atom sequence messages via a lock-free ring buffer during the audio thread. File path is sent once per change — the plugin holds state between cycles until explicitly reset.

Audio Pipeline

From Guitar to Speakers in Real Time

The full-duplex audio pipeline runs entirely in a dedicated background thread managed by Google Oboe. No UI thread involvement in the hot path.

  1. 01

    Oboe Captures Audio Input

    Google Oboe opens a full-duplex stream on the selected input device (USB interface, microphone, etc.) with the configured sample rate and burst size. The real buffer burst size is used to set the LV2 buf-size extension options at plugin instantiation time.

  2. 02

    LiveEffectEngine Processes Plugin Chain

    LiveEffectEngine::process() passes audio through each enabled plugin slot in order (Slot 1 → Slot 2 → Slot 3 → Slot 4). Bypassed plugins in the chain are skipped. The engine applies master gain scaling to the output buffer.

  3. 03

    Output Is Written to Device and File

    The processed audio buffer is written to the output device by Oboe. Simultaneously, if recording is active, the same buffer is passed to FileWriter which encodes and writes to the chosen format (WAV, FLAC, MP3, Opus, OGG) on device storage.

  4. 04

    UI Reflects Pedal and Engine State

    The Java UI layer communicates with the native engine via JNI calls from the UI thread. Parameter changes (slider moves, bypass toggles) are forwarded to the engine through safe JNI methods. The UI never blocks the audio thread.

Requirements

What You Need to Run & Build

Runtime Requirements

  • Android 12+ (API level 31)
  • ARM64-v8a, ARMv7, x86, or x86_64 ABI
  • RECORD_AUDIO permission
  • Any WDM-compatible USB audio interface

Build Requirements

  • Android Studio (latest stable)
  • Android SDK with API 31+ support
  • Android NDK (latest stable)
  • CMake 3.22.1 or higher
  • JDK 11+
  • Gradle 8.0+ (included wrapper)

Native Dependencies

  • Google Oboe v1.10.0
  • JACK Audio Connection Kit (libjack)
  • Lilv (LV2 plugin loader)
  • Sord (RDF store)
  • Sratom (Atom serialization)
  • libsndfile (WAV/FLAC I/O)
