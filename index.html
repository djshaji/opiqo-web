Capabilities

Everything You Need to Perform

Built on the industry-standard LV2 plugin format, Opiqo gives you studio-grade guitar processing anywhere.

LV2 Plugin Host

Full support for standard LV2 audio plugins. Compatible with thousands of open-source guitar effects, synths, and processors.

Real-time DSP

Low-latency audio processing using Oboe (Android), WASAPI (Windows), and JACK (Linux). Floating-point precision throughout.

4-Slot Effects Chain

Chain up to 4 LV2 plugins simultaneously. Independently bypass, reconfigure, or remove each slot in the signal path.

Multi-format Recording

Capture your processed signal post-FX in WAV, FLAC, MP3, Opus, or OGG Vorbis. Record while all 4 slots are active.

Preset Management

Save, export, and import your full pedalboard configuration as JSON. Restore any rig on any device in seconds.

Truly Cross-Platform

One shared DSP core across Android, Windows, and Linux. Same plugins, same presets, same workflow on every device you own.

Platforms

One App, Every Platform

Native performance and platform-appropriate UI on each operating system.

Android Edition

A full-featured LV2 host for Android 12+. Uses Google's Oboe library for the lowest possible audio latency on mobile hardware.

  • Google Oboe full-duplex audio (API 31+)
  • 54 bundled guitar effects plugins
  • Swipeable 4-pedal tab UI
  • Input & output device selection
  • Master volume slider
  • WAV, FLAC, MP3, Opus, OGG recording
  • Preset backup & restore via JSON
  • LV2 buf-size spec compliant
  • Multi-ABI: arm64-v8a, armeabi-v7a, x86, x86_64
Requirements: Android 12+ (API 31) · Android NDK · CMake 3.22.1+
Learn More →

Windows Edition

A native Win32 desktop host with WASAPI audio. Low-latency duplex streaming with a clean 2×2 plugin slot grid layout.

  • WASAPI shared & exclusive duplex streams
  • Win32 native UI — 2×2 plugin slot grid
  • MMCSS real-time audio thread priority
  • IMMDeviceEnumerator hot-plug detection
  • Power toggle state machine (Off / Starting / Running / Error)
  • WAV, MP3, OGG recording with quality selection
  • Settings dialog: sample rate, block size, share mode
  • Full preset import / export
  • MinGW cross-compile supported
Requirements: Windows 10+ · MinGW or MSVC · CMake 3.22+

Linux Edition

A native GTK4 desktop host using JACK Audio Connection Kit. Works on both classic JACK2 and modern PipeWire via the JACK compatibility layer.

  • JACK / PipeWire-JACK duplex audio
  • GTK4 UI — 2×2 plugin slot grid via GtkGrid
  • jack_set_port_registration_callback hot-plug
  • SCHED_FIFO real-time audio thread
  • Power toggle with JACK server-lost recovery
  • WAV, MP3, OGG recording
  • Settings dialog: JACK ports, buffer size (read-only from server)
  • XDG-compliant settings: ~/.config/opiqo/settings.json
  • X11 and Wayland compatible
Requirements: GTK4 ≥ 4.6 · JACK ≥ 1.9 · Lilv · libsndfile · CMake 3.22+
Workflow

Start Playing in Minutes

  1. 01

    Connect Your Guitar

    Plug in your guitar via a USB audio interface or use the device microphone. Select your input and output devices from the app.

  2. 02

    Load Your Plugins

    Tap + on any of the 4 pedal slots to browse and load an LV2 effect. Choose from 54 included plugins or your own.

  3. 03

    Tune Your Sound

    Adjust every plugin parameter in real time with labelled sliders. Bypass individual pedals to A/B compare your tone instantly.

  4. 04

    Record & Share

    Hit Record to capture your processed signal post-FX in WAV, FLAC, MP3, Opus, or OGG. Save presets and take your rig anywhere.

Get Opiqo

Download for Your Platform

Free and open source. Build from source or grab the latest release binary.

Android

  • Android 12+ (API level 31)
  • ARMv8, ARMv7, x86, x86_64
  • RECORD_AUDIO permission required
Get it on Google Play Learn More
Build from source 
git clone https://github.com/opiqo/opiqo.git
cd opiqo
./gradlew assembleRelease

Windows

  • Windows 10 or later (64-bit)
  • WASAPI (shared or exclusive mode)
  • Visual C++ Redistributable
Download Installer (.exe) Download ZIP (portable)
Build from source 
cmake --preset windows-default
cmake --build build-windows -j4

Linux

  • GTK4 ≥ 4.6
  • JACK ≥ 1.9 or PipeWire-JACK
  • Lilv, libsndfile
Download AppImage (coming soon) Flatpak (coming soon)
Build from source 
sudo apt install libgtk-4-dev libjack-jackd2-dev \
  liblilv-dev libsndfile1-dev cmake
cmake --preset linux-default
cmake --build build-linux -j$(nproc)
